A set of GET routes read the digest, never call a model, and never need a key: glossary listing and per-term lookup (by term or alias), section summaries optionally filtered by group, a single full section node, the overview and context, and a compressed archive of the whole tree. IDs containing slashes or hashes must be URL-encoded in the path, the archive route is the bulk transport used by MCP over an endpoint with a HEAD variant exposing the content-hash header so clients can skip downloads, and missing entries or unknown routes return a not-found JSON error.
