A support matrix across Astro, Docusaurus, VitePress, MkDocs, and plain static HTML, showing how each handles digest build, the keyword overlay, and agentic answers: Astro is turnkey through the integration while the others build the digest in their build, drop in the overlay, and use a hostable endpoint for answers. The CLI and MCP surfaces are host-neutral on every row, so a coding agent reads the same digest regardless of renderer.
