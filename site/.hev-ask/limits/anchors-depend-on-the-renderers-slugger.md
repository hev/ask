Deep links are only correct while the heading slugs hev ask generates match your renderer's id attributes; it uses github-slugger by default aligned with Astro and GitHub, each framework adapter declares its own scheme where that differs, and it ships a verify command that fails if any chunk anchor is missing from the built HTML, so wiring verify into CI catches a slugging change before a broken link ships.
