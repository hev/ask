Only the section summary, glossary, orientation context, and suggestions are model-authored while the structure, verbatim facts, overview, hashes, and anchors are derived in code, so the model only supplies the distillation, which is exactly the part worth doing in a skill. The recommended path is the bundled Claude Code skill running inside your existing subscription with no key or per-build token spend, and the fallback is the unattended build command for CI or non-skill users; either way the build is incremental and hash-gated, re-distilling only changed sections, and the tree is reviewed in pull requests.
