hev ask indexes sections rather than pages: each document is split on its headings to a configurable depth, content before the first heading becomes an intro chunk whose URL is the page itself, and each chunk carries its heading, cleaned prose, and a URL with an anchor generated by the same slugger the renderer uses so the link lands on a heading that exists. Each framework adapter declares its slug scheme, both code paths chunk through one shared function, and because the build reads files rather than a renderer the same digest comes out whatever framework renders the docs.
