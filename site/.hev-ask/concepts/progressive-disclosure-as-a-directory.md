Because the digest is a real directory, progressive disclosure is just the directory's own cost model, leaning on an agent's existing intuition that a listing is cheap and a full read may be large. The reads form a disclosure ladder where a listing reads frontmatter only (so the map can never leak a body and stays bounded by section count, safe to call speculatively and shallow by default) and every deeper rung is an explicit verb, so nothing larger than a title is ever returned by surprise; a coding agent climbs it with its own file tools while the CLI offers one verb per operation for keyless and remote cases.
