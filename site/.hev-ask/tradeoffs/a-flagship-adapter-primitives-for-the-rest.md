hev ask is host-neutral at the core but opinionated about Astro, whose integration is batteries-included (build, endpoint, and overlay from one config block) while every other framework wires two primitives itself: the static overlay, and, for agentic answers, the standalone endpoint. The upside is that the digest, overlay, CLI, and MCP are the same everywhere so adopting a new framework is an adapter not a fork and the keyword path is fully static; the cost off Astro is doing the wiring the integration would have done, since a turnkey plugin per framework is a goal not a guarantee.
