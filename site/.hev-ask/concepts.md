The core idea is that the digest is just a directory: your docs are distilled into a committed tree of one markdown file per section, read two ways from the same tree climbed progressively, by a coding agent using its own file tools and by the overlay reading server-side to synthesize a grounded answer for a human. The split that matters is that the tree is built offline with a strong model and committed to git while the readers run on demand, so no durable state lives in the running site.
