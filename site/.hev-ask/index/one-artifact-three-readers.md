The digest is built inside your coding agent (the bundled skill writes the tree using your existing subscription, with no key or per-build token spend), and once committed the same files serve three readers: the overlay for humans with instant keyword results plus a grounded answer on Enter and every result deep-linked, the CLI for agents to map, read, pull literals from, and search the tree keylessly from any shell, and the MCP server for agents with one tool that hydrates the whole tree to disk. hev ask only reads the content you point it at with no crawler, no external index, and nothing to keep in sync, and the committed tree is reviewable per section in every pull request.
