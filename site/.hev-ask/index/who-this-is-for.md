hev ask is for anyone building or maintaining a docs site whose content lives in Markdown or MDX on Astro, Docusaurus, VitePress, MkDocs, or anything rendering a folder of markdown, who wants search and answers that work without standing up a service or crawler, deep-link to the right section, answer questions in the reader's words rather than just matching keywords, and are queryable by a coding agent. Astro covers all of that in one integration; other frameworks drop in the static overlay and point it at an optional hosted endpoint for answers; and if you only need keyword search over a static site with no key, Pagefind is a simpler fit.
